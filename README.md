# Fairness and Explainability Under Missingness: An Interpretable Analysis of In-Hospital Mortality Prediction on the MIMIC-III Cohort

This repository contains the code used for a thesis studying how missing-data handling strategy jointly affects predictive performance, explainability, and fairness in an admission-time in-hospital mortality prediction model, using the MIMIC-III admissions cohort.

## Summary

- **Task:** Predict in-hospital mortality using only information available at the time of hospital admission (demographics, admission type/location, admitting diagnosis and procedure, insurance, religion, marital status). Post-admission summary variables (length of stay, lab/procedure counts, etc.) were deliberately excluded to avoid target leakage.
- **Missing-data strategies compared:** Mean imputation, median imputation, KNN imputation, MICE, MissForest, and XGBoost's native missing-value handling.
- **Models:** Logistic Regression, Random Forest, XGBoost, in a factorial design (each imputation strategy × each classifier).
- **Missingness experiment:** Synthetic missingness at 10%, 30%, and 50% under MCAR, MAR, and MNAR mechanisms.
- **Explainability:** SHAP (global importance, beeswarm, dependence plots, per-patient waterfall plots), with stability of explanations checked across imputation methods.
- **Fairness:** Demographic parity, equal opportunity, equalized odds, and false-negative-rate (FNR) gap, audited across gender, insurance type, age band, and ethnicity, with bootstrap 95% confidence intervals, and compared across all six missing-data strategies.

## Key results

| Metric | Value |
|---|---|
| Best model | Tuned XGBoost + MICE imputation |
| AUROC (95% CI) | 0.786 (0.773–0.799) |
| AUPRC | 0.339 |
| Sensitivity / Specificity | 0.700 / 0.729 |
| Largest fairness gap (FNR) | Age band: 0.515 (CI 0.390–0.649); Ethnicity: 0.505 (CI 0.380–1.000) |
| Smallest fairness gap (FNR) | Gender: 0.017 |

A key finding: no single missing-data strategy is uniformly fairest. Native missing-value handling produced the smallest fairness-gap point estimate for age band (0.315) but the largest for ethnicity (0.833) of any strategy tested, while mean imputation showed close to the opposite pattern. Bootstrap 95% confidence intervals are reported for every strategy-attribute combination; the age-band finding is reasonably well supported by non-overlapping intervals, while the ethnicity finding should be read as a consistent directional pattern rather than a pairwise-significant difference, given the wide, overlapping intervals typical of small ethnicity subgroups. To isolate the effect of imputation from hyperparameter tuning, MICE and native handling were also compared under an identical tuned configuration: MICE still significantly outperformed native handling (0.786 vs 0.755; bootstrap 95% CI for the difference [0.021, 0.040]; McNemar p<0.001), though by a smaller margin than an untuned comparison would suggest. This indicates that the choice of missing-data strategy should not be assumed to be fairness-neutral, and should be evaluated per protected attribute, with uncertainty reported alongside every point estimate.

## Data access

This study uses the **MIMIC-III** (Medical Information Mart for Intensive Care III) database. MIMIC-III is publicly available but requires:

1. Completing PhysioNet's human-subjects research training (CITI "Data or Specimens Only Research" course).
2. Signing a Data Use Agreement (DUA).
3. Requesting credentialed access via [https://physionet.org/content/mimiciii/](https://physionet.org/content/mimiciii/).

**Raw MIMIC-III data is not included in this repository**, in compliance with PhysioNet's data-sharing policy. To reproduce the results, obtain credentialed access to MIMIC-III and place the required data file in the same directory as the notebook (see the notebook's first data-loading cell for the exact expected filename) before running the notebook.

## How to run

1. Clone this repository:
   ```bash
   git clone https://github.com/wrim786-collab/mimic3-mortality-fairness-audit-.git
   cd mimic3-mortality-fairness-audit-
   ```

2. Create a virtual environment and install dependencies:
   ```bash
   python -m venv venv
   source venv/bin/activate        # on Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. Place your credentialed MIMIC-III data file in the repository's working directory (see `journal_paper.ipynb`'s first data-loading cell for the expected file name).

4. Launch Jupyter and run all cells in order:
   ```bash
   jupyter notebook journal_paper.ipynb
   ```
   Run cells top-to-bottom (Kernel → Restart & Run All) — later cells depend on variables created earlier (imputed datasets, trained models, fairness function).

## Reproducibility notes

- A fixed random seed (42) was used throughout for data splitting and model training; cross-validation was additionally repeated across seeds 42, 1, and 2 (15 folds total) to check stability.
- All preprocessing (imputation, scaling, class-weighting) was fitted on the training split only and applied to the test split, to avoid data leakage.
- Class imbalance (11.4% mortality) was addressed via balanced class weighting (`class_weight='balanced'` / `scale_pos_weight`), not synthetic resampling (e.g. SMOTE), to avoid any risk of resampling-related leakage across folds.
- Hyperparameters were tuned via `RandomizedSearchCV` (20 candidates, 3-fold stratified CV, optimising AUROC) on the MICE-imputed training data; the selected configuration was 300 estimators, max_depth 3, learning rate 0.1, subsample 1.0, and colsample_bytree 1.0 (also reported in the thesis Methodology chapter).
